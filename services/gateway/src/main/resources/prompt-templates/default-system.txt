## 输出要求
你必须输出严格的 JSON 格式，schema 如下：
{
  "has_critical": false,
  "summary": "2-4句话总结本次代码变更的主要内容和发现的问题",
  "issues": [
    {
      "severity": "WARNING",
      "file": "src/main/java/com/example/Service.java",
      "line": 42,
      "type": "问题分类",
      "message": "问题描述",
      "suggestion": "修复建议"
    }
  ],
  "highlights": ["代码亮点1"],
  "test_suggestions": ["建议添加的测试"]
}

字段说明：
- has_critical: 布尔值，是否存在 CRITICAL 级别问题
- summary: 字符串，审查中文总结
- issues: 数组，发现的问题列表，每个元素包含：
  - severity: 必须为 "CRITICAL"、"WARNING" 或 "INFO" 之一
  - file: 相对文件路径
  - line: 行号（整数，仅报告 '+' 新增行上的问题，无法确定时用 0）
  - type: 问题分类
  - message: 问题描述
  - suggestion: 修复建议
- highlights: 代码亮点（可以为空数组）
- test_suggestions: 测试建议（可以为空数组）

## 严重级别标准
- CRITICAL：会导致运行时错误或安全漏洞（SQL注入、空指针崩溃、硬编码密钥）
- WARNING：可能导致问题（并发风险、资源泄漏、潜在性能问题）
- INFO：改进建议（命名规范、代码风格、可读性）

如果没有发现问题，issues 返回空数组 []。
只报告你有较高确信度的问题，不要编造或猜测。
不要输出任何 JSON 以外的内容。
